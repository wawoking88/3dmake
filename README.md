# Scratch風 3Dゲームメーカー Blockly修正版

## 修正内容

- エラーが出たとき、画面上に実際のエラー文を表示
- Blocklyの読み込みを1本化して安定化
- TransformControlsを使わず、独自の簡易3D操作ガイドに変更
  - 赤線：X
  - 緑線：Y
  - 青線：Z
- 移動 / 向き変更 / スケール変更に対応
- ＋ボタンから、直方体・球・円柱・Capsule・円すいを追加
- 最初からCameraを追加
- 実行中はCameraの位置と向きが実際の画面になる
- Blocklyで、if文と「キーが押された」を分離
- キー表示は ↑ ↓ ← → に対応

## 起動方法

`index.html` を開いてください。

※ Three.jsとBlocklyをCDNから読むため、インターネット接続が必要です。

## 追加修正5

- ステージ上の操作ガイドを線だけから作り直し
  - 移動：赤/緑/青の矢印
  - スケール：赤/緑/青の四角つまみ

- 操作ガイドの当たり判定を子パーツまで拾うように修正
- スプライトの大きさに関係なく、操作ガイドが見やすいサイズになるように調整


## 追加修正6

- ステージの矢印・回転リング・スケールつまみをクリックしても、Cameraなど別スプライトが誤選択されないように修正
- 操作ガイドは「現在選択中のスプライト」だけを動かすように修正
- 操作中に選択対象がずれても、ドラッグ開始時のスプライトだけを編集するように修正


## 追加修正7

- 大きさをX/Y/Z別々に変更できるように修正
- 右側プロパティに「大X・大Y・大Z」を追加
- ステージ上のスケール操作も、赤=X、緑=Y、青=Zで個別変更
- Blocklyに「大きさ x y z にする」ブロックを追加


## 追加修正8

- ステージの矢印/つまみ/リング操作中に、カメラ操作や本体選択処理が混ざらないように修正
- ドラッグ開始時のスプライトIDを固定し、ドラッグ中はそのスプライトだけを編集
- pointerup / pointercancel / lostpointercapture で必ずドラッグ状態を解除
- つまみ操作時にOrbitControlsへイベントが流れないように、ステージ側で先に処理するよう修正


## 追加修正9

- 自作の移動/回転/スケールつまみをやめて、Three.js公式アドオンの TransformControls に置き換え
- タッチやドラッグの掴みっぱなし問題を軽減
- 移動/向き変更/スケールは公式UIで操作
- 実行中は TransformControls を非表示・解除


## 追加修正10

- TransformControlsのつまみを押した瞬間に、OrbitControls（カメラ操作）を先に停止
- つまみ上を押したときはスプライト選択処理を走らせないように修正
- TransformControlsのつまみサイズを少し大きくしてタッチしやすく調整
- canvas / stageWrap に touch-action:none を追加


## 追加修正11

- カメラ操作とスプライト編集を完全分離
- 移動 / 向き変更 / スケール中は OrbitControls を完全OFF
- カメラを動かしたい時だけ「視点操作」モードで OrbitControls をON
- 視点操作モード中は TransformControls を非表示にして、誤操作を防止
